Hard

這題需要搭配 Binary Indexed Tree（Fenwick Tree）進行巧妙的 prefix count 計算，是真的有夠難超 Hard。

2179. Count Good Triplets in an Array.cpp

Problem

You are given two permutations nums1 and nums2 of length n.

A good triplet is a set of indices (i, j, k) such that:

  • 0 <= i < j < k < n, and
  • the relative order of the values nums1[i], nums1[j], nums1[k] is the same as the relative order of nums2[i], nums2[j], nums2[k].

Return the total number of good triplets.

Example 1:

Input: nums1 = [0,1,2,3], nums2 = [2,0,1,3]
Output: 1
Explanation: Only the triplet (0,1,3) satisfies the conditions.

Example 2:

Input: nums1 = [4,0,1,3,2], nums2 = [4,1,0,2,3]
Output: 4

Constraints:

  • n == nums1.length == nums2.length
  • 3 <= n <= 10^5
  • 0 <= nums1[i], nums2[i] < n
  • nums1 and nums2 are permutations of 0 to n-1

解題思路

Step 1：重新定義題意

我們要找出三個 index (i, j, k)，在 nums1 和 nums2 中都符合「相對順序一致」的情況。
由於 nums1 和 nums2 都是 0~n-1 的排列，我們可以把其中一個 array 的值映射到另一個 array 中的位置，把問題轉換成一個 easier-to-check 的條件。

Step 2：將 nums1 映射到 nums2 的位置

我們把每個 nums1[i] 對應到 nums2 裡的位置，形成一個新的 index array idxMapping。
這樣我們只要在 idxMapping 中找出 嚴格遞增的三元組 (i, j, k)，就能表示原本 (nums1[i], nums1[j], nums1[k]) 和 (nums2[i], nums2[j], nums2[k]) 的相對順序一致。

Step 3：使用 BIT 計算 prefix 中的比自己小的個數

對於每個 val，我們可以這樣處理：

  • idxMapping[val] 表示 val 在 nums2 中的位置
  • 對於這個 pos，我們想知道在他左邊有幾個 index i 滿足 idxMapping[i] < pos → 這個用 BIT 來 query
  • 同樣地，我們也能知道右邊的數量是多少（補集扣掉就好）

Step 4：套用公式組合 triplet

只要知道：

  • 有多少 i 在左邊比當前 j 小（叫 left）
  • 有多少 k 在右邊比 j 大（叫 right）

那麼 (i, j, k) 的組合數就是 left * right
